## 桌面构图

- 导航约 68px 高，左右安全区约 24–36px。品牌在左，黑色菜单胶囊在右，导航只承担入口和品牌识别，不增加业务说明。
- 顶部能力 ticker 位于导航下方约 28px，单行、高度约 32px；内容足够轻，不与主标题竞争。
- 主标题居中、两行，视觉字号约 78–84px，行高约 1.02。实现中使用 Manrope 作为主体、Source Serif 4 Italic 强调 `traceable.`，不使用负字距。
- 中文说明紧跟标题，约 17px，宽度控制在 36–42 个中文字符以内。
- 登录框宽约 420–450px，位于视口水平中心；整体中心比画面几何中心略低，为标题、说明和 ticker 留出呼吸空间。
- 桌面曲线由两侧向中心包围，视觉稿中的线密度接近 20+20。实现保持 2.5px 描边和原始峰值透明度，曲线层覆盖在未激活面板之上。
- 流程索引从首屏底部露出，使用四至五列、细分隔线和大号序号，不使用独立卡片或阴影容器。

## 玻璃面板状态

- 视觉稿提供的是接近清晰态的结构参考：8px 圆角、细边框、低强度阴影、黑色全宽主按钮。
- 未激活态在实现中比视觉稿更透明：接近透明的白色背景、轻微模糊，输入边界和文字仍保持可读；层级低于弧线。
- 首次聚焦任意输入后，面板上移约 8px，层级越过弧线，背景变为高透明白色玻璃，并提高到 `blur(28px) saturate(140%)`。
- 浮现过渡约 420ms，激活后在当前页面会话保持清晰。
- 初始化、认证错误、提交中、加载和不可达状态直接使用清晰态，避免长表单和恢复操作被背景干扰。

## 背景与层级

- 桌面背景中央约 45% 是亮白低细节区；发票纸张、表格线和结构阴影集中在左右边缘，适合叠加标题和表单。
- 移动背景上半部保留更大留白，票据层从下半部两侧进入，避免导航和标题区域变脏。
- 背景不承担弧线动画；弧线由 CSS 独立渲染，保证数量、速度、响应式和减少动态偏好可控。
- 桌面层级顺序：背景 0、未激活面板 2、弧线 3、激活面板 4、导航 20、菜单 100。

## 移动折叠

- 810px 以下使用独立移动背景，不旋转桌面图。
- 侧边曲线隐藏，改为 40 条顶部横向嵌套曲线。曲线位于标题下方到面板上方的空间，并可穿过未激活面板表面。
- 标题约 46–50px，最多三行；中文说明居中，保持约 16px。
- 面板宽为视口减 32px，输入和按钮不因动效改变尺寸。
- 流程索引改为单列，只让前两项在首屏底部形成下一段内容提示。

## 色彩、边框与动效

- 背景：`#ffffff` 至冷白 `#f7f7f5`
- 主文字：`#0a0a0a`
- 次文字：`#6b6b6b`
- 细线：`rgba(10, 10, 10, 0.08)`
- 弧线：`#fcfaf8`
- 真实可用状态和焦点反馈可使用绿色 `#17c964`，其余区域保持黑白。
- 面板清晰态使用单层柔和阴影，不制造卡片套卡片。
- marquee 为 30s 线性循环；弧线为 5s 呼吸循环并按 0.25s 错开。
- `prefers-reduced-motion: reduce` 下停止循环和位移动画，保留静态代表状态。

## 实现修正

- 生成参考中的品牌符号不进入代码，正式品牌只显示 `Invoice OCR` 和 `发票识别与归档`。
- 生成参考中的票据文字只作为背景纹理；页面不展示可识别的真实金额、客户或公司标识。
- 生成参考用于尺度和层级，不替代无障碍语义、真实表单状态或现有认证契约。
