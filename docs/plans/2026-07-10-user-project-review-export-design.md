# 用户、项目、校对与导出扩展设计

日期：2026-07-10

## 1. 目标

在现有 Invoice OCR MVP 基础上补齐以下产品闭环：

- 以品牌化登录落地页作为应用首页。
- 支持首位管理员初始化和管理员管理后续用户。
- 使用固定的普通用户、财务、管理员三级角色。
- 使用共享项目和私有项目组织发票，并提供系统默认“未分类”项目。
- 完成待校对工作队列、明细校正保存和导出记录页面。
- 将 Dashboard 静态指标替换为真实业务数据。

本设计保留现有 FastAPI、SQLAlchemy、Alembic、React、Vite、Celery、PostgreSQL 和 Redis 架构，不引入 Workspace 或可编辑 RBAC 权限矩阵。

## 2. 角色与权限

### 普通用户

- 查看和管理自己上传的发票。
- 创建、编辑和归档自己的私有项目。
- 查看并选择有效共享项目。
- 校对、确认、归档和导出自己有权访问的发票。

### 财务用户

- 跨用户查看、校对和导出发票。
- 查看全部项目，包括其他用户的私有项目。
- 创建和管理共享项目。
- 不管理用户、OCR 系统设置或其他管理员能力。

### 管理员

- 包含财务用户的全部能力。
- 创建用户、修改角色、启停账号和重置密码。
- 管理全部共享及私有项目。
- 管理 OCR 运营商、凭据和额度设置。

系统禁止禁用或降级最后一个有效管理员。

## 3. 认证与初始化

### 初始化状态

应用增加系统状态记录。首次迁移时创建单例状态行；已有用户的数据库视为已初始化。

`GET /api/v1/auth/bootstrap-status` 返回系统是否已经初始化。未初始化时，前端显示首位管理员创建表单。

`POST /api/v1/auth/bootstrap` 在锁定系统状态行的事务中再次检查用户数量，创建首位管理员并标记系统已初始化。并发请求只能有一个成功。

初始化完成后关闭公开注册。其他用户只能由管理员创建，创建时默认角色为 `user`。

### 会话

用户增加 `session_version`。会话令牌包含版本号；用户修改密码或管理员重置密码时递增版本，使旧会话立即失效。禁用账号后，现有会话在下一次请求时被拒绝。

前端增加顶层认证边界，统一处理：

- 初始化状态加载。
- 首位管理员创建。
- 登录、退出和会话恢复。
- 401 返回登录页。
- 403 展示禁用或权限不足状态。

### 用户管理 API

- `GET /api/v1/admin/users`
- `POST /api/v1/admin/users`
- `PATCH /api/v1/admin/users/{user_id}`
- `POST /api/v1/admin/users/{user_id}/reset-password`
- `PATCH /api/v1/auth/password`

用户、角色、状态和密码重置操作写入审计日志。API 永不返回密码或密码哈希。

## 4. 项目模型

新增 `projects` 表：

- `id`
- `name`
- `description`
- `visibility`: `private`、`shared`、`system`
- `status`: `active`、`archived`
- `system_key`: 系统项目使用，普通项目为空
- `created_by`
- `created_at`
- `updated_at`
- `archived_at`

系统创建唯一的 `system_key=uncategorized` 项目，名称为“未分类”。该项目不可编辑、归档或删除。

普通用户只能创建私有项目。财务和管理员可以创建共享项目；管理员可以管理全部项目。私有项目名称在创建者范围内唯一，共享项目名称在共享范围内唯一。

项目使用归档而非物理删除。归档项目保留历史发票和导出记录，但不能用于新上传或发票移动目标。

### 发票关系

`invoice_documents` 增加非空 `project_id` 外键。迁移创建“未分类”后，将现有文档全部回填到该项目，再设置非空约束。

项目在上传时确定；未传项目时使用“未分类”。发票列表、详情、导出和 Dashboard 返回项目摘要。

普通用户可以把自己的发票移动到共享项目或自己的有效私有项目；财务和管理员可以移动其有权访问的发票。

### 项目 API

- `GET /api/v1/projects`
- `POST /api/v1/projects`
- `PATCH /api/v1/projects/{project_id}`
- `POST /api/v1/projects/{project_id}/archive`
- `POST /api/v1/projects/{project_id}/restore`

## 5. 前端体验

### 登录落地页

未认证首页使用全屏品牌场景图，体现发票归档、扫描和数字化处理用途。品牌标题和用途说明直接叠加在场景上，登录或初始化表单使用紧凑、清晰的操作面板。移动端保持表单优先，背景不影响文字对比度。

系统未初始化时显示“创建首位管理员”；初始化完成后显示登录。认证成功后进入工作台。

### 工作台导航

顶部账号菜单显示姓名、角色、修改密码和退出。导航增加“项目”；管理员额外看到“用户管理”。

### 项目管理

项目页面按共享项目、我的私有项目和已归档组织。共享/私有状态同时使用文字与图标表达。普通用户只能创建私有项目；财务和管理员可按权限创建共享项目。

### 发票库和上传

发票库增加项目侧栏、项目筛选和项目列；移动端使用项目下拉筛选。上传页增加项目选择，默认“未分类”，共享和私有项目分组显示。详情页允许移动项目。

### 待校对

待校对页面使用三个标签：

- 待人工确认：`needs_review`
- 疑似重复：`duplicate_suspected`
- 识别失败：`failed` 或关联 OCR 最终失败

页面展示高密度工作表格，突出缺失字段、OCR 错误、重复风险和项目。支持进入详情、确认无误、重新识别，并仅在没有明显错误或重复风险时允许批量确认。

实现 `PUT /api/v1/invoices/{invoice_id}/items`，确保明细校正持久化并记录修正审计。

### 导出记录

页面顶部提供导出创建区，支持项目、日期、状态、格式、明细和 OCR 元数据选项。任务表展示创建人、范围、格式、状态、创建时间、过期时间和下载操作。

运行中的任务自动刷新；失败任务显示安全错误；过期任务提供基于原筛选条件重新创建导出的入口。

### Dashboard

新增 `GET /api/v1/dashboard/summary`，提供待校对数量、识别失败数量、近 30 天确认金额、OCR 队列数量和最近导出。前端不再显示静态指标。

## 6. 后端扩展

发票列表增加 `project_id` 和财务/管理员可用的 `uploaded_by` 筛选。发票 PATCH 支持 `project_id`，并执行目标项目可见性与有效状态校验。

导出筛选增加项目和上传用户条件；导出行和元数据包含项目。`export_tasks` 增加安全的 `error_message`，Celery 失败时记录业务错误状态而不暴露内部堆栈或文件路径。

所有新增用户、项目、发票移动、校正、批量确认和导出操作写入审计日志。

## 7. 错误处理

新增稳定错误码，包括：

- `AUTH_BOOTSTRAP_COMPLETE`
- `AUTH_LAST_ADMIN_REQUIRED`
- `USER_EMAIL_EXISTS`
- `PROJECT_NOT_FOUND`
- `PROJECT_FORBIDDEN`
- `PROJECT_ARCHIVED`
- `PROJECT_NAME_EXISTS`
- `PROJECT_SYSTEM_IMMUTABLE`
- `INVOICE_ITEMS_INVALID`

前端按错误码显示中文操作建议。内部异常、数据库路径、堆栈和密钥不得返回客户端或写入可见错误字段。

## 8. 测试与验收

后端按 TDD 覆盖初始化、会话版本、用户管理、最后管理员保护、项目权限、迁移回填、发票项目过滤、明细保存、校对批量操作、Dashboard 和项目导出。

前端测试覆盖认证状态、角色导航、项目分组、状态映射和导出状态。Docker E2E 覆盖首位管理员、用户创建、共享/私有项目、上传、OCR、校对、导出和持久化。

迁移必须在 PostgreSQL 上从当前版本升级验证。最终执行完整后端测试、前端测试与构建、Docker E2E、`linux/amd64` 镜像构建、健康检查以及桌面/移动浏览器视觉检查。
