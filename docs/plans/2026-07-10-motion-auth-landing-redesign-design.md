# Motion Auth Landing Redesign Design

日期：2026-07-10

状态：已确认

## 目标

将当前认证入口从明亮档案式双栏页面重做为沉浸式单页落地页。视觉语言参考 MotionSites 的极简创意机构 Hero，但所有内容、资产和交互都改为发票 OCR、人工校对、项目归档、权限管理和私有部署语义。

新页面必须继续承载以下现有行为：

- 登录已初始化系统
- 创建首位管理员
- 显示认证加载状态
- 显示服务不可达状态并重试
- 保持现有认证错误、密码显示切换、自动填充和表单校验契约
- 登录成功后进入现有工作台，不修改工作台导航和业务页面

本设计取代 `docs/design/auth-landing-direction-2.md` 的视觉方向，但不废弃其中的认证功能、安全和无障碍要求。

## 设计判断

页面是私有发票系统的入口，不是虚构客户背书或销售转化页面。参考站的构图、动态曲线、跑马灯、字体反差和全屏菜单可以保留，但创意机构套餐、预约头像、合作公司和无依据指标必须删除。

- 视觉方向：黑白极简、沉浸式、轻编辑感、动态纸张与扫描光轨
- 主题：固定明亮主题
- 主色：黑、白和冷灰
- 语义绿：只用于真实可用状态和焦点反馈
- 动效强度：高，但只作用于背景曲线、跑马灯、菜单和登录框层级变化
- 页面密度：低，保留大面积留白
- 页面结构：Hero、能力带、流程索引、页脚

## 固定导航

- 固定在视口顶部，层级为页面常规内容之上、全屏菜单之下
- 桌面水平内边距 36px，内容最大宽度 1200px
- 移动水平内边距 20px
- 左侧品牌使用无衬线和衬线斜体的组合字标
- 右侧使用黑色 `Menu` 胶囊按钮和 Lucide `ChevronUp`
- 菜单按钮文字保持单行，具备明确悬停、按下和键盘焦点状态

## 全屏菜单

菜单使用白色全屏覆盖层，400ms 淡入和淡出。中央入口为：

1. 登录系统
2. 原件归档与 OCR
3. 人工校对
4. 项目与权限
5. 私有部署

交互要求：

- “登录系统”关闭菜单并聚焦邮箱输入框；初始化状态聚焦姓名输入框
- 其他入口滚动到页面中的真实内容锚点
- `Escape` 关闭菜单
- 打开后焦点进入菜单并约束在抽屉内
- 关闭后焦点返回 Menu 按钮
- 页面背景在菜单打开时禁止滚动
- 页脚显示项目名和版权信息，不显示版本号或虚构状态

## Hero 构图

- 登录状态 Hero 最小高度约 900px
- 首管理员初始化状态根据字段高度扩展到约 1040px 或更高
- 顶部导航下方依次为 ticker、标题、说明和中央登录框
- 登录框处于页面几何中心，标题位于登录框上方，不与登录框重叠
- Hero 底部使用渐进模糊过渡到白色能力区域
- 标题最大两行，说明保持简短

## 项目专属背景图

不使用用户提供的第三方图片 URL，不依赖外部图片 CDN。生成两张本地 WebP：

- `frontend/public/auth-motion-hero-desktop.webp`
- `frontend/public/auth-motion-hero-mobile.webp`

### 桌面图内容

横向冷白发票纸层、半透明票据边缘、深黑结构阴影和一条柔和的 OCR 扫描光轨。中央区域为标题和登录框保留低细节空间。图像不出现可读文字、金额、货币符号、人物、手、电脑、办公桌、键盘或 UI。

### 移动图内容

纵向票据层叠和扫描光轨，中央保留更大的低细节区域。它是独立生成的移动构图，不裁剪或旋转桌面图。

## 多层弧线脉冲

- 桌面左侧 20 条、右侧 20 条
- 使用 `2.5px solid #FCFAF8` 描边
- 左侧去除左边框并在右侧形成大圆角
- 右侧去除右边框并在左侧形成大圆角
- 每条宽度从 60px 开始递增 10px
- 动画延迟为索引乘以 0.25s
- `line-pulse` 动画时长 5s，保持接近参考站的密度、描边和峰值透明度
- 动画容器必须使用 `pointer-events: none`
- 移动端隐藏侧边曲线，改为顶部 40 条横向曲线，保持完整密度

## 玻璃登录框

登录框是页面的主要交互层，具有未激活和已激活两个视觉状态。

### 未激活状态

- 位于曲线动画层下方
- 背景接近透明
- 使用较轻的背景模糊
- 边框、标签、输入边界和按钮必须清晰可识别
- 弧线可以从登录框前方经过
- 登录框仍可点击和键盘聚焦

### 首次聚焦后的状态

用户首次点击或键盘聚焦任意输入框后：

- 登录框提升到弧线动画之上
- 向上移动约 8px
- 背景逐渐变为高透明白色玻璃
- `backdrop-filter` 增强到约 `blur(28px) saturate(140%)`
- 显示更明确的边框、高光和柔和阴影
- 过渡时长约 420ms
- 激活状态在本次页面会话中保持，不因字段切换、提交或焦点离开而退回

### 强制清晰状态

以下状态直接使用已激活玻璃层：

- 首管理员初始化
- 认证错误
- 提交中
- 加载状态
- 服务不可达状态

这样可以保证长表单、错误说明和恢复操作始终可读。

## 表单内容

现有字段、属性和行为保持不变：

- 登录：邮箱、密码、显示密码、登录按钮、管理员帮助说明
- 初始化：姓名、邮箱、密码、确认密码、密码要求、管理员权限说明、创建按钮
- 密码自动填充属性保持正确
- 每个输入继续使用可见标签
- 错误继续使用 `role="alert"`
- 加载状态继续使用 `aria-live="polite"` 和 `aria-busy`
- 不添加记住我、忘记密码、公开注册或社交登录

## 跑马灯

顶部 ticker 和 Hero 下方能力带使用无缝横向循环：

- 30s 线性循环
- 内容复制足够次数以避免空白
- 两端使用透明遮罩
- 只移动内部轨道，不改变布局尺寸
- 项目能力使用简短文本，不使用虚构公司 Logo

## 流程索引

能力带下方使用极简流程索引，不使用卡片：

1. 上传原件
2. OCR 提取
3. 人工校对
4. 项目归档
5. 结构化导出

每项由大号标题、一句说明和细分隔线组成。桌面可使用五列或不对称网格，移动端改为单列。所有文案必须来自系统已实现能力。

## 字体

- 本地托管现代无衬线字体
- 本地托管 `Source Serif 4` normal 和 italic
- 中文回退使用 `PingFang SC`、`Microsoft YaHei` 和通用无衬线
- 不在运行时请求 Google Fonts
- 页面不依赖外部字体 CDN
- 字体文件使用 `font-display: swap`
- 字号通过明确断点调整，不依赖连续视口缩放

## 响应式

### 1200px 以下

- Hero 水平内边距降为 32px
- 标题字号使用中等桌面规格
- 菜单链接字号缩小
- 登录框保持中央轴线

### 810px 以下

- 使用移动背景图
- Hero 水平内边距 20px 至 24px
- 侧边曲线替换为顶部 40 条横向曲线
- 标题保持两行以内
- 登录框宽度为视口减 32px，最大宽度不超过桌面规格
- 初始化状态增加 Hero 高度，不压缩字段
- 能力带和流程索引改为纵向结构
- 菜单链接使用移动字号

### 验收宽度

- 375px
- 768px
- 1024px
- 1440px

所有宽度必须满足：

- 页面级 `scrollWidth === innerWidth`
- 标题、表单和按钮不重叠
- 登录按钮完整可见
- 菜单不会超出视口
- 背景图已加载且构图没有空白区域

## 动效与性能

- 只为弧线、跑马灯、菜单和登录框层级变化提供持续或状态动画
- DOM 动画只使用 `transform` 和 `opacity`
- 不使用 `window` scroll 事件
- 不使用 GSAP
- 曲线和跑马灯使用 CSS 动画
- 菜单与登录框状态由 React 控制，动画仍由 CSS 完成
- `prefers-reduced-motion: reduce` 下停止曲线和跑马灯，菜单和玻璃层立即切换
- 背景图片固定宽高或覆盖区域，避免 CLS
- 背景资源压缩为适合首屏加载的 WebP

## 无障碍

- 初始透明登录框仍达到 WCAG AA
- 所有输入标签可见且与输入关联
- 密码按钮保持 44px 触控尺寸
- Menu 按钮和关闭按钮具备可访问名称
- 全屏菜单使用合适的对话框或导航语义
- 焦点不会进入菜单背后的页面
- 减少动画偏好得到尊重
- 弧线和背景图为装饰内容，不进入无障碍树
- 状态不能只依靠颜色表达

## 测试与验收

### 组件测试

- 登录状态渲染新标题、ticker、菜单和登录表单
- 初始化状态渲染完整管理员字段
- 首次聚焦后登录框进入并保持 engaged 状态
- 菜单打开、关闭、Escape 和焦点返回
- 错误、加载和服务不可达状态直接使用清晰玻璃层
- 不出现记住我、忘记密码、注册或虚构客户内容

### 浏览器 QA

- 登录页、初始化页、加载页和错误页
- 桌面与移动背景图加载
- 弧线数量、方向和动画层级
- 登录框点击前后层级与玻璃变化
- 菜单焦点约束和滚动锁定
- 375、768、1024、1440px 无溢出
- `prefers-reduced-motion` 下内容可读
- 控制台无错误或警告

## 明确不做

- 不修改认证 API
- 不修改登录后的工作台设计
- 不使用第三方背景图和头像
- 不展示虚构合作公司
- 不展示无法验证的指标
- 不添加账号注册、找回密码或社交登录
- 不将表单文字栅格化到背景图
