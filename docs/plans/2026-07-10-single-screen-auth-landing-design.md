# Single-Screen Auth Landing Design

日期：2026-07-10

状态：已确认

## 目标

将当前认证首页收紧为桌面单屏页面，同时删除不再需要的辅助文案和菜单交互。桌面端不出现纵向滚动条；810px 以下移动端继续使用现有可滚动布局。

## 删除内容

- 删除能力带左侧文案“围绕企业财务流程构建”。
- 删除右上角 `Menu` 按钮。
- 删除全屏菜单、菜单链接、焦点陷阱、滚动锁定和对应跳转逻辑。
- 删除相关 React 状态、事件监听、Lucide 图标和 CSS。

## 桌面单屏结构

桌面认证首页保持以下内容，并全部放入 `100dvh`：

1. 左上角 `Invoice OCR` 品牌
2. 顶部能力 ticker
3. Hero 标题和说明
4. 居中登录玻璃框
5. 能力 marquee
6. 五步流程索引
7. 页脚

页面使用四行网格组织：Hero、能力带、流程索引、页脚。Hero 占用剩余空间，其余三行使用紧凑固定内容高度。桌面普通登录和加载/错误状态禁止页面纵向滚动。

## 尺寸调整

- 桌面标题缩至约 56–60px。
- 缩小 Hero 顶部留白、标题与说明间距、说明与登录框间距。
- 登录框保持 420px 最大宽度，但缩小内边距、字段高度、表单间距和帮助说明间距。
- 能力 marquee 删除左侧标签后占满内容宽度，高度控制在约 38–44px。
- 流程索引保留五列，但缩小标题、序号、图标、说明和上下内边距。
- 页脚压缩为单行细边界区域。

## 响应式

- `> 810px`：页面高度为 `100dvh`，`overflow: hidden`，不出现滚动条。
- `<= 810px`：恢复自然文档流和纵向滚动，保留独立移动背景、40 条顶部弧线及单列流程。
- 首管理员初始化态字段更多，桌面允许自然滚动，避免压缩表单可用性。

## 保留行为

- 登录框未聚焦和聚焦后的玻璃层级变化不变。
- 登录、首管理员初始化、加载、错误、密码显示和表单验证逻辑不变。
- 20+20 桌面弧线、40 条移动弧线和跑马灯动效不变。
- 已登录工作台完全不变。

## 验收标准

- 页面中不存在 `Menu`、首页导航弹层或“围绕企业财务流程构建”。
- `1440×900` 和 `2048×982` 下，普通登录页 `scrollHeight === innerHeight`。
- 桌面标题、表单、能力带、五步流程和页脚均在同一视口内可见。
- `375px` 和 `768px` 下仍可纵向滚动且无横向溢出。
- 登录框聚焦后仍变为 `blur(28px) saturate(140%)` 的清晰玻璃态。
